=== Внешние интеграции
==== Внешняя система аутентификации (IAM)
- **Интеграция с IAM** (OAuth2, Keycloak)
- **Функциональность**: аутентификация и авторизация пользователей
- **Протоколы**: OAuth 2.0, OpenID Connect
- **Схема взаимодействия**:
  - API Gateway направляет запросы аутентификации в внешнюю систему IAM
  - Внешняя система проверяет учетные данные пользователей
  - Возвращает токены доступа для дальнейшей работы с системой
  - Система мониторинга также взаимодействует с IAM для получения метрик безопасности

==== Платежная система
- **Интеграция с платежной системой**
- **Функциональность**: обработка платежей за услуги системы
- **Протоколы**: REST API, Webhook
- **Схема взаимодействия**:
  - Микросервис обработки запросов инициирует платеж через API платежной системы
  - Получает уведомления о статусе платежа через Webhook
  - Синхронизирует статусы платежей с внутренней базой данных

==== Система уведомлений
- **Интеграция с системой уведомлений**
- **Функциональность**: отправка уведомлений пользователям (SMS, email, push и т.д.)
- **Протоколы**: REST API, SMTP
- **Схема взаимодействия**:
  - Микросервис обработки запросов формирует сообщения уведомлений
  - Передает их в систему уведомлений через API
  - Получает статусы доставки для последующей обработки
