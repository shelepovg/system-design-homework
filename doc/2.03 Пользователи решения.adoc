=== Пользователи решения
==== Внутренние пользователи:
. Администраторы системы - управляют системой, мониторят производительность, управляют конфигурациями
. Поддержка - взаимодействует с внутренними пользователями для решения технических вопросов и поддержания системы в рабочем состоянии
. Работники бизнеса - используют систему для выполнения бизнес-процессов и аналитики
. Маркетинговые специалисты - используют систему для управления кампаниями и анализом результатов

==== Внешние пользователи:
. Конечные пользователи - используют интерфейс для отправки запросов
. Внешние системы - интегрируются через API или другие протоколы
    .. Уведомления - система отправки уведомлений пользователям через различные каналы (SMS, email, push-уведомления). Интегрируется через REST API и SMTP для доставки сообщений пользователям о статусах операций, важных событиях и системных уведомлениях
    .. Платежные системы - для обработки финансовых транзакций пользователей. Интегрируется через REST API и Webhook для инициации платежей, получения статусов транзакций и синхронизации финансовых данных
    .. Аутентификация - внешняя система управления идентификацией и доступом (IAM), обеспечивающая аутентификацию и авторизацию пользователей. Интегрируется через протоколы OAuth 2.0 и OpenID Connect (например, Keycloak) для проверки учетных данных и выдачи токенов доступа
