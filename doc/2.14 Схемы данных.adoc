=== Схемы данных

Документ указывает расположение и формат схем данных решения. Полное описание модели данных (таблицы, домены, связи, Database per Service) — документ 2.06. Система хранения (технологии, масштабирование, бэкапы) — документ 2.13.

*Где находятся схемы:*

  * *Концептуальная и логическая модель:* 2.06 «Модель данных» — перечень сущностей, атрибуты, типы, ограничения, разбиение по БД (users_db, requests_db, payments_db, notifications_db, audit_db).
  * *DBML (визуализация и генерация DDL):* каталог `dbml/`, файл `model.dbml` — схема в формате DBML для просмотра в DBML CLI / dbdiagram.io и при необходимости экспорта в SQL. 

*Использование:* при изменении структуры данных правки вносятся в 2.06 и при необходимости синхронизируются с `dbml/model.dbml`; 2.14 служит точкой входа к этим артефактам.
